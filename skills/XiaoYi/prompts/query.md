## 问题构造模板

### 标准问题模板
```
[问题背景]：简要描述当前开发场景或遇到的问题
[具体问题]：详细说明需要解决的问题
[已尝试方案]：列出已经尝试的解决方案（可选）
[期望结果]：描述希望达到的效果
```

### 深度查询模板 (--deep)
```
[问题背景]：[必填] 详细的开发环境和场景描述
[核心问题]：[必填] 需要深入分析的技术难点
[技术栈]：[必填] 相关技术组件和版本
[约束条件]：[选填] 性能、兼容性等要求
[已尝试方案]：[必填] 已有的解决方案及结果
[期望深度]：原理分析 / 最佳实践 / 架构建议
```

### 思维链查询模板 (--thinking)
```
[问题场景]：描述具体的使用场景
[分析需求]：希望AI展示推理过程的问题
[关注点]：特别关注哪些方面的分析
```

## 查询优化建议

### 1. 明确性原则
- ✅ 使用具体的技术术语
- ✅ 提供版本信息
- ✅ 说明运行环境
- ❌ 避免模糊表述

### 2. 上下文完整性
- ✅ 提供必要的代码片段
- ✅ 说明错误信息
- ✅ 描述预期行为
- ❌ 避免孤立提问

### 3. 问题聚焦
- ✅ 一次提问一个核心问题
- ✅ 分层次展开细节
- ❌ 避免多个不相关问题混杂

### 4. 格式规范
- ✅ 使用Markdown格式
- ✅ 代码块标注语言
- ✅ 关键信息加粗
- ❌ 避免无格式大段文本

## 注意事项

1. **网络要求**：确保能访问华为AI服务
2. **问题长度**：建议问题不超过500字
3. **敏感信息**：不要在问题中包含密钥、密码等敏感信息
4. **响应时间**：深度查询可能需要更长时间
5. **结果验证**：AI生成的代码需要实际测试验证
